  What the Router Does

  The router is a fast, cheap LLM call (or even a structured output call) with a small prompt:

  Given a user query about sales data, classify which screen tab should answer it:

  - "multi" ‚Äî Query needs data from 2+ tabs (e.g., "give me a full summary")
  - "clarify" ‚Äî Ambiguous query, need to ask user

  Return: { tab, reason }

  - Conversation-aware: Router sees history so "what about Europe?" after an APAC question routes to the same tab
